## 产品愿景
**成为制造型企业首选的智能采购决策平台**

---

## 一、执行摘要

本产品旨在为制造型企业提供智能化的工业品采购解决方案。通过 AI 技术自动识别采购需求，实时对比多家工业品电商平台的商品价格、交货期和最小起订量，以推荐卡片形式展示比价结果。用户确认后自动生成采购申请，经二级审批后传输至企业 ERP 系统。

**核心价值**：解决工业品采购价格不透明、比价耗时的痛点，预计帮助客户实现采购成本降低 10%，采购效率提升 30%。

---

## 二、问题陈述

### 2.1 当前痛点

- **价格不透明**：工业品品类繁多，不同供应商价格差异大，采购人员难以获取最优价格
- **比价耗时**：需要手动登录多个电商平台逐一查询，效率低下
- **流程繁琐**：采购申请需线下流转，审批后手动录入 ERP，耗时易出错

### 2.2 解决方案

构建智能采购系统，实现：
1. 多渠道价格自动抓取与对比
2. AI 智能识别采购需求（支持文本、规格、图片、PDF）
3. 自动化采购申请与审批流程
4. 一键对接企业 ERP 系统

### 2.3 业务影响

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 采购成本降低 | 10% | 对比系统上线前后同一品类采购价格 |
| 采购效率提升 | 30% | 采购周期从平均 X 天缩短至 Y 天 |

## 四、用户故事与验收标准（详细拆解）

### 故事 1：采购需求输入

**作为** 采购专员
**我想** 通过多种方式（文本、规格、图片、PDF）输入采购需求
**以便** 系统能自动识别并匹配商品

#### 动作拆解

```
1. 文本输入模式
   1.1 用户在搜索框输入产品名称/关键词
   1.2 系统进行模糊匹配（支持同义词、别名）
   1.3 返回匹配的商品列表（包含名称、规格、图片）
   1.4 用户选择正确商品 → 进入比价流程

2. 结构化输入模式
   2.1 用户填写：产品名称 + 规格型号 + 采购数量
   2.2 系统解析规格型号，匹配商品库
   2.3 返回精确匹配结果

3. 图片上传模式（二期）
   3.1 用户上传产品图片
   3.2 系统进行 OCR 识别 + 图像特征提取
   3.3 匹配商品库，返回候选列表

4. PDF 规格书上传模式（二期）
   4.1 用户上传 PDF 规格书
   4.2 系统提取文本内容
   4.3 NLP 实体识别（产品名、规格、数量）
   4.4 返回识别结果供用户确认
   4.5 确认后进入比价流程
```

#### 验收标准

- [ ] 支持输入产品名称（模糊匹配，匹配度 > 80%）
- [ ] 支持输入规格型号、数量
- [ ] 支持上传产品图片，系统自动识别（二期）
- [ ] 支持上传 PDF 规格书，系统自动提取（二期）
- [ ] 需求输入后 3 秒内返回识别结果
- [ ] 识别结果不确定时，返回 Top 5 候选商品供选择
- [ ] 输入信息过少时，提示"请补充更多产品信息"

### 故事 2：AI 智能比价

**作为** 采购专员
**我想** 系统自动对比多家电商平台的价格、交货期、最小起订量
**以便** 快速找到最优采购方案

#### 动作拆解

```
1. 触发比价
   1.1 用户确认商品后，点击"开始比价"
   1.2 系统同时向 3 家平台发起查询请求

2. 并行查询
   2.1 震坤行 API 查询 → 返回商品信息
   2.2 西域 API 查询 → 返回商品信息
   2.3 京东工业品 API 查询 → 返回商品信息
   2.4 任意平台超时（5秒），标记为"获取失败"

3. 数据聚合
   3.1 汇总各平台返回的商品信息
   3.2 统一数据格式（名称、规格、单价、库存、起订量、交货期）
   3.3 价格数据存入缓存（有效期 1 小时）

4. 结果排序
   4.1 综合最优：价格×0.5 + 交货期×0.3 + 起订量×0.2 加权得分
   4.2 价格最低：按单价升序
   4.3 交货最快：按交货期升序
   4.4 起订量最低：按 MOQ 升序

5. 异常处理
   5.1 平台无货：显示"暂无货"，灰色展示
   5.2 平台无价格：显示"暂无报价"
   5.3 平台接口异常：显示"获取失败"，允许重试
```

#### 验收标准

- [ ] 支持同时查询震坤行、西域、京东工业品三家平台
- [ ] 返回结果包含：商品图片、规格型号、供应商、库存状态、预计交货时间、最小起订量
- [ ] 支持按价格、交货期、最小起订量排序
- [ ] 数据缓存机制，确保价格时效性
- [ ] 平台无货或无价格时，标注显示"暂无报价"
- [ ] 单个平台查询超时（5秒）不影响其他平台展示
- [ ] 比价过程中显示加载状态（"正在查询 X 平台..."）

#### 遗漏检查

- [ ] 是否需要展示价格趋势？→ 建议：历史价格走势图（ nice to have）
- [ ] 是否需要区分含税/不含税价格？→ 建议：明确标注
- [ ] 是否有假货/正品保障？→ 建议：可增加供应商信用评级

### 故事 5：二级审批流程

**作为** 部门经理
**我想** 审批采购申请，同意后自动推送至 ERP
**以便** 简化审批流程，提高效率

#### 动作拆解

```
1. 一级审批（部门经理）
   1.1 部门经理收到审批通知（站内信 + 邮件）
   1.2 登录系统，进入"待审批"列表
   1.3 查看采购申请详情：
       - 申请人、申请时间
       - 商品明细及金额
       - 供应商信息
       - 预算使用情况（如有）
   1.4 审批操作：
       - 同意 → 提交至二级审批
       - 驳回 → 填写驳回原因 → 退回申请人
   1.5 可选择批量审批（多选后批量同意/驳回）

2. 二级审批（ERP 执行）
   1.1 一级审批通过后，自动进入 ERP 创建流程
   1.2 系统调用用友 ERP 接口，创建采购订单
   1.3 返回 ERP 订单号，关联本地采购申请
   1.4 状态变更：已审批 → 已推送 ERP

3. 审批超时处理
   3.1 一级审批超时（默认 24 小时）→ 自动催办
   3.2 超时 48 小时 → 升级通知上级

4. 驳回处理
   4.1 驳回时必须填写原因（必填）
   4.2 申请人收到驳回通知
   4.3 申请可修改后重新提交
```

#### 验收标准

- [ ] 一级审批：采购专员提交 → 部门经理审批
- [ ] 二级审批：部门经理审批 → ERP 系统
- [ ] 审批通过后，自动创建采购单并推送至用友 ERP
- [ ] 审批驳回时，需填写驳回原因
- [ ] 支持站内信和邮件通知
- [ ] 审批超时自动催办
- [ ] 驳回后可修改重新提交

#### 遗漏检查

- [ ] 是否需要审批加签？→ 建议：特殊金额需要多级审批
- [ ] 是否需要审批委托？→ 建议：支持委托他人审批
- [ ] 是否需要审批统计？→ 建议：审批时效统计报表

### 故事 7：价格缓存与更新（补充）

**作为** 系统
**我想** 通过缓存机制保证价格时效性，同时减少 API 调用
**以便** 平衡性能与数据准确性

#### 动作拆解

```

3. 价格异常检测
   3.1 单价变动超过 20% → 标记"价格波动较大"
   3.2 变动超过 50% → 标记"价格异常"并预警
```

#### 验收标准

- [ ] 价格缓存有效期 1 小时
- [ ] 展示价格更新时间
- [ ] 支持手动刷新价格
- [ ] 价格异常波动标记提醒

   2.2 按钮、标签、提示信息等全部翻译
   2.3 保持用户操作习惯一致

3. 通知翻译
   3.1 邮件通知模板支持多语言
   3.2 站内信通知支持多语言

- [ ] 语言切换实时生效
- [ ] 翻译覆盖所有界面文案

## 五、功能需求

### 5.1 核心功能

#### 功能 1：智能需求识别
- **描述**：支持多种方式输入采购需求，AI 自动识别商品信息
- **输入方式**：文本（产品名称、关键词）、结构化（规格型号、数量）、图片（OCR 识别）、文档（PDF 规格书）
- **边界情况**：输入信息过少时提示补充；识别结果不确定时返回多个候选

#### 功能 2：多平台价格抓取
- **描述**：对接震坤行、西域、京东工业品三家电商平台
- **数据字段**：商品名称、规格型号、单价、库存、起订量、预计交货时间
- **缓存策略**：价格数据缓存 1 小时，库存数据实时查询
- **边界情况**：平台无货显示"暂无货"；无价格显示"暂无报价"；接口异常标记"获取失败"

#### 功能 3：智能推荐排序
- **排序维度**：综合最优、价格最低、交货最快、起订量最低

#### 功能 4：采购申请生成
- **申请单内容**：商品明细、供应商信息、预计交货时间、付款方式、申请人、申请时间

#### 功能 5：ERP 对接
- **对接方式**：用友 API / 中间库
- **传输数据**：采购单号、商品明细、供应商、总金额、审批状态

### 5.2 非功能范围
- 第一版仅支持用友 ERP，金蝶为二期规划
- 第一版仅支持震坤行、西域、京东工业品

## 六、技术约束

### 6.1 性能要求
- 需求识别响应时间：< 3 秒
- 价格查询响应时间：< 5 秒
- 页面加载时间：< 2 秒
- 支持 100 并发用户（单租户），支持横向扩展

### 6.2 安全要求
- 用户身份认证（SSO / 账号密码）
- 采购数据加密传输
- 敏感操作日志审计

### 6.3 多租户架构

**设计目标**：支持多家企业客户共用系统，数据完全隔离

| 租户隔离层级 | 实现方式 |
|--------------|----------|
| 数据隔离 | 租户 ID 字段，每张表包含 tenant_id |
| 配置隔离 | 租户独立配置（logo、主题、审批流程） |
| API 隔离 | 租户标识 Token 解析 |
| 缓存隔离 | Redis Key 包含租户前缀 |
| 日志隔离 | 日志记录包含租户 ID |

**租户配置项：**
- 企业名称、Logo、Favicon
- 主题配色（品牌色）
- 审批流程配置（审批人、审批层级）
- 对接 ERP 信息（不同企业不同 ERP 实例）
- 电商平台 API 授权（可选不同平台组合）
- 语言设置（中文/英文）

**租户管理功能：**
- 超级管理员后台
- 租户开通/禁用
- 租户配置管理
- 租户使用统计

### 6.4 国际化（i18n）支持

**支持语言**：中文（zh-CN）、英文（en-US）

**翻译范围：**
- 界面文案（按钮、标签、提示信息）
- 错误信息
- 邮件通知模板
- 报表导出

**实现方式：**
- 前端：i18n 方案（Vue i18n / React i18next）
- 后端：消息 KEY 国际化
- 数据库：多语言配置表

**语言切换：**
- 用户个人设置中选择语言
- 租户级别默认语言配置
- 支持实时切换无需重启

### 6.5 扩展性设计

**模块化设计：**
- 采购需求模块（独立服务）
- 比价引擎模块（独立服务）
- 审批流程模块（独立服务）
- ERP 对接模块（独立服务）

**配置化驱动：**
