### 2.3 比价引擎模块

#### 页面清单
| 页面 | 路径 | 功能描述 |
|------|------|----------|
| 比价结果页 | /purchase/compare | 卡片展示比价结果 |

#### 功能点
- [ ] F3.1 三平台并行查询（震坤行/西域/京东工业品）
- [ ] F3.2 熔断器实现（连续3次失败自动熔断30秒）
- [ ] F3.3 价格缓存（有效期1小时）
- [ ] F3.4 强制刷新（防刷：5分钟最多3次）
- [ ] F3.5 价格异常检测（>20%标记，>50%预警）
- [ ] F3.6 综合最优算法：Score = P×0.5 + D×0.3 + M×0.2

#### API 接口
```
POST /api/v1/prices/compare         # 发起比价
GET  /api/v1/prices/result/{task_id} # 获取比价结果
POST /api/v1/prices/refresh         # 强制刷新价格
GET  /api/v1/prices/cache/{sku}    # 获取缓存价格
```

#### 平台适配器
- [ ] A1 震坤行适配器（API接入）
- [ ] A2 西域适配器（API接入）
- [ ] A3 京东工业品适配器（API接入）

### 2.5 采购申请生成模块

#### 页面清单
| 页面 | 路径 | 功能描述 |
|------|------|----------|
| 申请确认页 | /purchase/confirm | 确认商品、补充信息 |
| 申请详情页 | /purchase/{id} | 申请详情、状态查看 |

#### 功能点
- [ ] F5.1 价格时效性校验（生成申请前强制校验，超过5%提示确认）
- [ ] F5.2 确认清单展示（商品、数量、供应商、单价、交期、运费、合计）
- [ ] F5.3 补充信息（需求部门、项目名称、用途说明）
- [ ] F5.4 付款方式选择（预付/月结/货到付款）
- [ ] F5.5 交货地点选择
- [ ] F5.6 采购申请单生成（编号：PR-YYYYMMDD-XXX）
- [ ] F5.7 申请前预览
- [ ] F5.8 审批前可撤回

#### API 接口
```
POST /api/v1/purchase/orders               # 创建采购申请
GET  /api/v1/purchase/orders               # 采购申请列表
GET  /api/v1/purchase/orders/{id}          # 采购申请详情
PUT  /api/v1/purchase/orders/{id}          # 更新采购申请
POST /api/v1/purchase/orders/{id}/submit   # 提交审批
POST /api/v1/purchase/orders/{id}/withdraw # 撤回申请
```

## 四、代码规范与审核标准

### 4.1 开发规范
- 遵循 Git Flow 工作流
- 分支命名：`feature/功能名`、`bugfix/问题描述`、`hotfix/紧急修复`
- Commit 信息规范：`[模块] 描述`
- 代码Review必须通过才能合并

### 4.2 审核清单
- [ ] 代码规范检查通过
- [ ] 单元测试覆盖率 > 70%
- [ ] API 文档已更新
- [ ] 无安全漏洞（SQL注入、XSS等）
- [ ] 性能符合要求（响应时间 < 5秒）

### 4.3 测试标准
- 功能测试：所有验收标准必须通过
- 接口测试：所有 API 必须测试
- 集成测试：跨模块流程必须通过
- 性能测试：满足响应时间要求
