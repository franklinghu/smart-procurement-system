## 一、执行摘要

本产品需求文档定义了一个面向全公司的项目管理系统的核心需求。该系统旨在解决当前公司项目管理中存在的任务跟踪混乱、工时统计不准确、审批流程不规范等问题。

系统目标用户包括：管理层（报表查看）、项目经理（任务管理）、普通员工（工时填报）。核心功能包括：任务管理、进度追踪、工时统计、文档协作、审批流（项目立项）。采用SaaS云端部署，Web端访问。

预期价值：提升工时统计准确度至95%以上，简化项目立项审批流程，实现项目进展可视化。

---

## 二、问题陈述

### 2.1 当前状况

- **任务管理分散**：任务信息散落在邮件、文档中，缺乏统一管理
- **进度不透明**：项目进度依赖人工汇报，无法实时掌握
- **工时统计粗放**：员工工时记录方式不统一，数据准确性低
- **审批效率低**：项目立项审批依赖线下流程，耗时长

### 2.2 解决方案

构建统一的Web端项目管理系统，实现：
- 任务的全生命周期管理
- 实时进度可视化追踪
- 规范的工时填报与统计
- 在线审批流程

### 2.3 业务影响

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 工时统计准确度 | ~60% | ≥95% |
| 项目立项审批周期 | 3-5天 | ≤1天 |
| 任务状态更新及时性 | 滞后 | 实时 |

## 四、用户画像

### 4.1 管理层

- **角色**：公司高管、部门负责人
- **目标**：查看公司/部门项目整体进展、汇总报表
- **痛点**：无法快速了解项目状态，报表获取依赖人工
- **技术能力**：中高级

### 4.2 项目经理

- **角色**：项目负责人
- **目标**：创建/分配任务、跟踪进度、管理项目文档、发起立项审批
- **痛点**：任务分发依赖会议和邮件，进度汇报繁琐
- **技术能力**：中高级

### 4.3 普通员工

- **角色**：项目参与者
- **目标**：查看个人任务、填报工时、上传项目文档
- **痛点**：需要登录多个系统，工时填报流程复杂
- **技术能力**：初级至中级

## 五、用户故事与验收标准

### 5.1 任务管理

**故事1：创建任务**

- **作为** 项目经理
- **我希望** 创建任务并分配给团队成员
- **以便** 明确责任分工，跟踪任务进度

**验收标准：**
- [ ] 可创建任务，填写任务标题、描述、截止日期
- [ ] 可选择任务负责人
- [ ] 可设置任务优先级（高/中/低）
- [ ] 任务创建成功后，相关人员收到通知

**故事2：任务状态更新**

- **作为** 任务负责人
- **我希望** 更新任务状态
- **以便** 记录任务进展

**验收标准：**
- [ ] 可将任务状态更新为：待处理/进行中/已完成/已取消
- [ ] 状态变更记录可追溯
- [ ] 状态变更通知任务相关人员

### 5.2 进度追踪

**故事3：项目进度看板**

- **作为** 项目经理
- **我希望** 查看项目进度看板
- **以便** 快速了解项目整体进展

**验收标准：**
- [ ] 显示项目任务完成百分比
- [ ] 显示各阶段任务数量和状态分布
- [ ] 支持按项目/按人员筛选

**故事4：进度报表**

- **作为** 管理层
- **我希望** 查看公司/部门级项目报表
- **以便** 了解整体项目状况

**验收标准：**
- [ ] 显示项目数量统计（进行中/已完成/延期）
- [ ] 显示任务完成率趋势图
- [ ] 支持导出报表

### 5.3 工时统计

**故事5：工时填报**

- **作为** 普通员工
- **我希望** 每日填报工时
- **以便** 记录项目工作量

**验收标准：**
- [ ] 可选择日期和项目填写工时
- [ ] 可填写工时说明
- [ ] 当日可修改，隔日不可修改
- [ ] 工时提交后通知项目经理

**故事6：工时统计查看**

- **作为** 项目经理/管理层
- **我希望** 查看工时统计数据
- **以便** 了解项目投入情况

**验收标准：**
- [ ] 可按项目/人员/时间段查看工时汇总
- [ ] 显示工时明细列表
- [ ] 支持导出Excel

### 5.4 文档协作

**故事7：项目文档上传**

- **作为** 项目经理/团队成员
- **我希望** 上传项目相关文档
- **以便** 项目成员共享资料

**验收标准：**
- [ ] 支持上传常见格式（Word/Excel/PPT/PDF/图片）
- [ ] 文件大小限制100MB
- [ ] 文档按项目归类
- [ ] 可在线预览常用格式

**故事8：文档版本管理**

- **作为** 项目经理
- **我希望** 文档保留历史版本
- **以便** 需要时查看历史内容

**验收标准：**
- [ ] 文档更新自动生成新版本
- [ ] 可查看历史版本列表
- [ ] 可下载历史版本

### 5.5 审批流

**故事9：项目立项审批**

- **作为** 项目经理
- **我希望** 发起项目立项审批
- **以便** 获得项目正式立项

**审批流程：**
```
提交申请 → 部门负责人审批 → 分管领导审批（可选）→ 通过/驳回
```

**验收标准：**
- [ ] 可填写立项申请，包括：项目名称、项目目标、成员、计划时间、预算
- [ ] 审批人可选择（从组织架构中选择）
- [ ] 审批人可批准或驳回，填写审批意见
- [ ] 审批状态实时可查
- [ ] 审批完成后自动通知申请人

## 六、功能需求

### 6.1 核心功能

#### 功能1：任务管理

- **描述**：创建、分配、更新任务
- **用户流程**：
  1. 项目经理创建任务
  2. 设置任务详情和负责人
  3. 系统通知负责人
  4. 负责人更新任务状态
- **边界情况**：
  - 任务负责人离职：支持重新分配
  - 项目取消：任务自动标记为已取消
- **错误处理**：
  - 网络异常：本地缓存，联网后同步
  - 必填项缺失：前端拦截提示

#### 功能2：进度追踪

- **描述**：实时展示项目进度和统计报表
- **用户流程**：
  1. 选择查看维度（项目/人员）
  2. 查看进度看板/报表
  3. 如需详情，点击进入具体项目
- **边界情况**：
  - 无项目数据：显示空状态引导页
  - 数据加载慢：显示骨架屏

#### 功能3：工时统计

- **描述**：员工填报工时，管理层查看统计
- **用户流程**：
  1. 员工选择日期和项目
  2. 填写工时和说明
  3. 提交工时
  4. 项目经理/管理层查看汇总
- **边界情况**：
  - 重复填报：当日可覆盖
  - 加班工时：区分正常工作时间和加班时间
- **错误处理**：
  - 超时未填报：次晨提醒

#### 功能4：文档协作

- **描述**：项目文档上传、预览、版本管理
- **用户流程**：
  1. 进入项目文档区
  2. 上传文件或创建在线文档
  3. 文档自动保存到项目
- **边界情况**：
  - 文件类型不支持：提示不支持并列出支持类型
  - 文件过大：提示压缩或分卷上传

#### 功能5：审批流

- **描述**：项目立项在线审批
- **用户流程**：
  1. 项目经理填写立项申请
  2. 提交审批
  3. 审批人处理（批准/驳回）
  4. 申请人收到通知
- **边界情况**：
  - 审批人外出：支持转交
  - 审批超时：自动催办
- **错误处理**：
  - 审批人不可用：提示选择其他审批人

### 6.2 系统功能

| 功能 | 描述 |
|------|------|
| 用户管理 | 支持邮箱/手机号注册登录，角色权限管理 |
| 组织架构 | 支持部门/团队管理，用户归属 |
| 消息通知 | 系统内消息+邮件通知 |
| 数据安全 | HTTPS传输，敏感数据加密 |

### 6.3 非功能性需求

- **性能**：页面加载≤2秒，API响应≤500ms
- **可用性**：支持Chrome/Edge/Safari/Firefox最新版本
- **安全**：满足等保基本要求，密码加密存储

### 6.4 不包含范围

- 移动端App（Web响应式设计）
- 财务/薪资系统集成
- 复杂的项目资源调配
- 甘特图功能
- 知识库/百科功能

## 七、技术约束

### 7.1 性能要求

- 单页面加载时间 ≤ 2秒
- API平均响应时间 ≤ 500ms
- 支持同时在线用户 500+

### 7.2 安全要求

- 传输加密：全站HTTPS
- 密码存储：加盐哈希
- 权限控制：基于角色的访问控制（RBAC）
- 数据备份：每日自动备份，保留30天

### 7.3 技术栈

- 前端：React + TypeScript
- 后端：Node.js/Python
- 数据库：PostgreSQL + Redis
- 文件存储：对象存储（OSS/MinIO）
- 部署：Docker + Kubernetes（云端）

### 7.4 集成

- 暂不与外部系统集成
- 飞书/钉钉通知（后期版本可考虑）
