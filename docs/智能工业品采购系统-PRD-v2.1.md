# 智能工业品采购系统 PRD

## 智能工业品采购系统 PRD（详细版）

### 一、执行摘要

本产品旨在为制造型企业提供智能化的工业品采购解决方案。通过 AI 技术自动识别采购需求，实时对比多家工业品电商平台的商品价格、交货期和最小起订量，以推荐卡片形式展示比价结果。用户确认后自动生成采购申请，经二级审批后传输至企业 ERP 系统。

**核心价值**：解决工业品采购价格不透明、比价耗时的痛点，预计帮助客户实现采购成本降低 10%，采购效率提升 30%。

---

### 二、问题陈述

#### 2.1 当前痛点
- 价格不透明：工业品品类繁多，不同供应商价格差异大
- 比价耗时：需要手动登录多个电商平台逐一查询
- 流程繁琐：采购申请需线下流转，审批后手动录入 ERP

#### 2.2 业务影响

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 采购成本降低 | 10% | 对比上线前后同一品类采购价格 |
| 采购效率提升 | 30% | 采购周期缩短比例 |

### 四、用户故事与验收标准（详细拆解）

#### 故事 1：采购需求输入

**动作拆解：**

1. **文本输入模式**
   - 用户在搜索框输入产品名称/关键词
   - 系统进行模糊匹配（支持同义词、别名）
   - 返回匹配的商品列表
   - 用户选择正确商品 → 进入比价流程

2. **结构化输入模式**
   - 用户填写：产品名称 + 规格型号 + 采购数量
   - 系统解析规格型号，匹配商品库
   - 返回精确匹配结果

3. **图片上传模式（二期）**
   - 用户上传产品图片 → OCR 识别 + 图像特征提取
   - 匹配商品库，返回候选列表

4. **PDF 规格书上传模式（二期）**
   - 用户上传 PDF 规格书
   - 系统提取文本内容 → NLP 实体识别
   - 返回识别结果供用户确认

**验收标准：**
- 支持输入产品名称（模糊匹配，匹配度 > 80%）
- 支持输入规格型号、数量
- 需求输入后 3 秒内返回识别结果
- 识别结果不确定时，返回 Top 5 候选商品供选择
- 输入信息过少时，提示"请补充更多产品信息"

**遗漏检查：**
- 是否支持历史采购记录自动填充？→ 建议：支持从历史记录快速选择
- 是否支持批量导入？→ 建议：Excel 批量导入采购清单

#### 故事 2：AI 智能比价

**动作拆解：**

1. **触发比价**
   - 用户确认商品后，点击"开始比价"
   - 系统同时向 3 家平台发起查询请求

2. **并行查询**
   - 震坤行 API 查询 → 返回商品信息
   - 西域 API 查询 → 返回商品信息
   - 京东工业品 API 查询 → 返回商品信息
   - 任意平台超时（5秒），标记为"获取失败"

3. **数据聚合**
   - 汇总各平台返回的商品信息
   - 统一数据格式（名称、规格、单价、库存、起订量、交货期）
   - 价格数据存入缓存（有效期 1 小时）

4. **结果排序**
   - 综合最优：价格×0.5 + 交货期×0.3 + 起订量×0.2 加权得分
   - 价格最低：按单价升序
   - 交货最快：按交货期升序
   - 起订量最低：按 MOQ 升序

5. **异常处理**
   - 平台无货：显示"暂无货"，灰色展示
   - 平台无价格：显示"暂无报价"
   - 平台接口异常：显示"获取失败"，允许重试

**验收标准：**
- 支持同时查询震坤行、西域、京东工业品三家平台
- 返回结果包含：商品图片、规格型号、供应商、库存状态、预计交货时间、最小起订量
- 支持按价格、交货期、最小起订量排序
- 数据缓存机制，确保价格时效性
- 单个平台查询超时（5秒）不影响其他平台展示

#### 故事 3：推荐卡片展示

**动作拆解：**

1. **卡片渲染**
   - 按排序规则生成商品卡片列表
   - 每张卡片包含：商品图片、名称、规格、供应商、价格、库存、交货时间、最小起订量

3. **排序切换**
   - 顶部提供排序切换标签：综合最优 | 价格最低 | 交货最快 | 起订量最低
   - 切换时无需重新请求，直接本地排序

**验收标准：**
- 每个商品以卡片形式展示
- 卡片包含：商品图片、名称、规格、供应商、价格、库存、预计交货时间、最小起订量
- 支持按综合最优、价格最低、交货最快排序

#### 故事 4：采购申请生成

**动作拆解：**

1. **确认商品**
   - 用户点击"生成采购申请"

   - 用户点击"提交审批"

4. **撤销操作**
   - 审批前可撤回申请
   - 撤回后商品信息保留，可重新编辑提交

**验收标准：**
- 用户选择推荐方案后，一键生成采购申请
- 采购申请包含：商品信息、数量、供应商、单价、交货期、总价
- 采购申请自动提交至审批流程

- 提交前可预览完整申请单

#### 故事 5：二级审批流程

**动作拆解：**

1. **一级审批（部门经理）**
   - 部门经理收到审批通知（站内信 + 邮件）
   - 登录系统，进入"待审批"列表
   - 查看采购申请详情
   - 审批操作：同意 → 提交至二级审批；驳回 → 填写原因 → 退回申请人

2. **二级审批（ERP 执行）**
   - 一级审批通过后，自动进入 ERP 创建流程
   - 系统调用用友 ERP 接口，创建采购订单
   - 返回 ERP 订单号，关联本地采购申请
   - 状态变更：已审批 → 已推送 ERP

3. **审批超时处理**
   - 一级审批超时（默认 24 小时）→ 自动催办
   - 超时 48 小时 → 升级通知上级

4. **驳回处理**
   - 驳回时必须填写原因（必填）
   - 申请人收到驳回通知
   - 申请可修改后重新提交

**验收标准：**
- 一级审批：采购专员提交 → 部门经理审批
- 二级审批：部门经理审批 → ERP 系统
- 审批通过后，自动创建采购单并推送至用友 ERP
- 审批驳回时，需填写驳回原因
- 支持站内信和邮件通知
- 审批超时自动催办

### 十九、商业可行性评估

#### 19.1 市场切入点与竞争力分析

**高频刚需**
- 品类繁多：工业品（MRO）采购具有品类繁多、单价低、比价难的典型痛点
- AI 自动化：通过 AI 自动识别和多平台 API 实时比价，切中了制造型企业提升效率的"七寸"
- 市场规模：中国 MRO 采购市场规模达数万亿级别，存在巨大数字化转型需求

**差异化优势**
- 聚合器定位：相比于单一电商平台（如震坤行或京东），本系统作为"聚合器"提供了中立的比价视角
- 合规审计友好：对于需要内控合规的审计流程极具吸引力，支持比价数据留痕与追溯

#### 19.2 盈利模式可行性

**SaaS 收费模式**
- 分层策略：采用按用户/月收费及企业定制化的分层策略（基础版免费，专业版 999 元/用户/月，企业版定制报价）
- ROI 支撑：宣称可降低 10% 采购成本。对于年采购额 1000 万的企业，节省 100 万足以覆盖软件支出

**收入预测**

| 年度 | 客户数量 | ARR（万元） | 增长率 |
|------|----------|-------------|--------|
| 第1年 | 50 家 | 300 | - |
| 第2年 | 120 家 | 900 | 200% |
| 第3年 | 300 家 | 2400 | 167% |

#### 19.3 风险与壁垒

**数据护城河风险**
- 核心依赖：产品核心功能依赖电商平台 API，若平台方限制抓取或提高接口费用，将受影响
- 缓解策略：建立私有化商品库（已在二期规划中），通过历史订单积累形成自己的价格基准

**ERP 对接成本**
- 复杂度：工业企业 ERP 版本极其碎片化，实际交付时的实施人力成本可能是盈利的"杀手"
- 缓解策略：采用适配器模式，预留缓冲时间，分阶段对接

### 二十、评审建议与细节优化

#### 20.1 业务逻辑完善（PM 视角）

##### 20.1.1 价格时效性闭环

**问题**：当前缓存机制为 1 小时，但工业品价格波动大，生成采购申请后价格可能已变动

**解决方案**：
- 在用户点击"生成采购申请"的前一步，强制触发一次实时价格校验
- 如果价格变动超过阈值（如 5%），弹窗提示用户确认新价格
- 确认后更新缓存，再进入申请确认流程

**验收标准**：
- 采购申请提交前必须完成实时价格校验
- 价格变动超过 5% 时强制提示用户确认
- 校验后的价格有效期为 10 分钟，超时需重新校验

##### 20.1.2 运费逻辑（落地价 Landed Cost）

**问题**：工业品通常涉及重货、批量采购，运费是不可忽略的成本

**解决方案**：
- 比价引擎增加运费模板配置，支持按重量、体积、配送距离计算运费
- 在比价结果中增加"落地价"（Landed Cost）展示：商品价格 + 运费 + 税费
- 落地价 = 商品价格 + 运费模板计算结果

**验收标准**：
- 支持配置不同供应商的运费计算规则
- 比价卡片展示商品价格和落地价两种价格
- 落地价可按价格、交货期、最小起订量排序

##### 20.1.3 供应商评价体系

**问题**：当前仅展示价格，缺少供应商质量评估维度

**解决方案**：
- 在商品卡片中增加供应商评分维度展示：
  - **历史交付准时率**：过去 30 天按时交付比例
  - **售后评分**：基于历史采购的售后服务评价（1-5 星）
  - **正品保障**：是否为平台认证商家
- 供应商评分数据来源于历史订单数据统计

**验收标准**：
- 卡片展示供应商历史交付准时率（如 95%）
- 卡片展示供应商售后评分（如 4.8/5.0）
- 支持按综合评分排序

#### 20.2 系统架构强化（Architect 视角）

##### 20.2.1 比价引擎并发优化：熔断器模式（Circuit Breaker）

**背景**：当前三家平台并行查询，超时 5 秒。如果某个平台 API 不稳定，会影响整体响应

**解决方案**：
- 引入熔断器模式，当某个平台 API 连续失败超过阈值（如 3 次），自动"熔断"
- 熔断期间，该平台数据不再请求，直接返回"获取失败"或使用缓存数据
- 每隔一定时间（如 30 秒）尝试一次"半开"请求，如果成功则恢复

**熔断器配置**：
```
failureThreshold: 3 (连续失败 3 次后熔断)
timeout: 30s (熔断后 30 秒尝试半开)
successThreshold: 1 (半开成功后立即恢复)
```

**验收标准**：
- 单个平台 API 异常不影响其他平台展示
- 熔断后系统自动降级，不阻塞用户操作
- 熔断器状态可监控（正常/熔断/半开）

##### 20.2.2 多租户安全增强

**问题**：当前采用 tenant_id 行级隔离，对于大型企业客户，数据合规要求更高

**解决方案**：
- 针对"企业版"客户，提供数据库 Schema 级别隔离选项
- 每个企业版客户拥有独立的数据库 Schema，物理隔离更彻底
- 支持配置化切换：普通客户用行级隔离，企业客户用 Schema 隔离

**架构调整**：
```
普通租户（共享数据库）→ tenants 表 tenant_id 过滤
企业版租户（独立 Schema）→ 独立数据库实例，Schema 隔离
```

**验收标准**：
- 支持租户级别选择隔离模式
- 企业版客户数据完全物理隔离
- 租户配置界面可切换隔离模式

#### 20.3 核心模块增补

##### 20.3.1 比价算法加权公式

为了让"综合最优"更科学，明确权重算法：

**公式**：
```
Score = P_norm × 0.5 + D_norm × 0.3 + M_norm × 0.2
```

其中：
- **P_norm**：归一化价格得分（价格越低得分越高）
- **D_norm**：归一化交期得分（交货期越短得分越高）
- **M_norm**：归一化起订量得分（MOQ 越低得分越高）

**归一化方法**：
```
P_norm = (max_price - current_price) / (max_price - min_price)
D_norm = (max_delivery - current_delivery) / (max_delivery - min_delivery)
M_norm = (max_moq - current_moq) / (max_moq - min_moq)
```

**验收标准**：
- 综合最优分数明确展示
- 权重可配置（后台调整权重参数）
- 支持按不同排序维度展示

##### 20.3.2 审批流引擎细节完善

**条件审批：金额触发器**
- 单笔金额 > 5000 元 → 自动加签至财务总监
- 单笔金额 > 20000 元 → 加签至总经理
- 可配置不同金额阈值的审批人

**合规存证：数字签名**
- 审批通过时，系统自动对当时的比价快照进行"数字签名"
- 签名内容：商品信息 + 价格 + 供应商 + 审批时间 + 审批人
- 签名结果存储，确保证据链不可篡改，应对审计

**验收标准**：
- 支持按金额自动加签
- 审批记录包含完整比价快照
- 快照数据可追溯、不可篡改
