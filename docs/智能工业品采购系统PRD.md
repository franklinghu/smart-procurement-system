## 一、执行摘要

本产品旨在为制造型企业提供智能化的工业品采购解决方案。通过 AI 技术自动识别采购需求，实时对比多家工业品电商平台的商品价格、交货期和最小起订量，以推荐卡片形式展示比价结果。用户确认后自动生成采购申请，经二级审批后传输至企业 ERP 系统。

**核心价值**：解决工业品采购价格不透明、比价耗时的痛点，预计帮助客户实现采购成本降低 10%，采购效率提升 30%。

---

## 二、问题陈述

### 2.1 当前痛点

- **价格不透明**：工业品品类繁多，不同供应商价格差异大，采购人员难以获取最优价格
- **比价耗时**：需要手动登录多个电商平台逐一查询，效率低下
- **流程繁琐**：采购申请需线下流转，审批后手动录入 ERP，耗时易出错

### 2.2 解决方案

构建智能采购系统，实现：
1. 多渠道价格自动抓取与对比
2. AI 智能识别采购需求（支持文本、规格、图片、PDF）
3. 自动化采购申请与审批流程
4. 一键对接企业 ERP 系统

### 2.3 业务影响

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 采购成本降低 | 10% | 对比系统上线前后同一品类采购价格 |
| 采购效率提升 | 30% | 采购周期从平均 X 天缩短至 Y 天 |

## 五、功能需求

### 5.1 核心功能

#### 功能 1：智能需求识别

- **描述**：支持多种方式输入采购需求，AI 自动识别商品信息
- **输入方式**：
  - 文本：产品名称、关键词
  - 结构化：规格型号、数量
  - 图片：产品照片，OCR 识别
  - 文档：PDF 规格书，文本提取
- **处理逻辑**：NLP 实体识别 → 商品库匹配 → 返回候选商品
- **边界情况**：
  - 输入信息过少时，提示补充
  - 识别结果不确定时，返回多个候选

#### 功能 2：多平台价格抓取

- **描述**：对接震坤行、西域、京东工业品三家电商平台，获取商品价格
- **数据源**：
  - 震坤行 API
  - 西域 API
  - 京东工业品 API
- **数据字段**：商品名称、规格型号、单价、库存、起订量、预计交货时间
- **缓存策略**：
  - 价格数据缓存 1 小时
  - 库存数据实时查询
- **边界情况**：
  - 平台无货：显示"暂无货"
  - 平台无价格：显示"暂无报价"
  - 平台接口异常：标记为"获取失败"

#### 功能 3：智能推荐排序

- **描述**：根据用户偏好，智能排序推荐商品
- **排序维度**：
  - 综合最优（价格 + 交货期 + 起订量加权）
  - 价格最低
  - 交货最快
  - 起订量最低
- **推荐逻辑**：根据历史采购记录学习用户偏好，自动调整排序权重

#### 功能 4：采购申请生成

- **描述**：用户确认方案后，自动生成采购申请单
- **申请单内容**：
  - 商品明细（名称、规格、数量、单价）
  - 供应商信息
  - 预计交货时间
  - 付款方式
  - 申请人、申请时间
- **审批流程**：部门经理 → ERP

#### 功能 5：ERP 对接

- **描述**：审批通过后，自动推送采购单至用友 ERP
- **对接方式**：用友 API / 中间库
- **传输数据**：采购单号、商品明细、供应商、总金额、审批状态

### 5.2 非功能范围

- 第一版仅支持用友 ERP，金蝶为二期规划
- 第一版仅支持震坤行、西域、京东工业品
- 怡合达、米思米为后续扩展

## 六、技术约束

### 6.1 性能要求

- 需求识别响应时间：< 3 秒
- 价格查询响应时间：< 5 秒
- 页面加载时间：< 2 秒
- 支持 100 并发用户

### 6.2 安全要求

- 用户身份认证（SSO / 账号密码）
